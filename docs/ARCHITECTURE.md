## Genel Bakış

**VetApp** küçük veteriner klinikleri için modernize edilmiş yönetim sistemi. Randevu, müşteri, hayvan, sağlık kaydı, fiyat talebi ve WhatsApp entegrasyonunu yönetir.

### Tech Stack
- **Frontend**: Next.js 14 (App Router), TypeScript, React 18
- **Backend**: Node.js (Next.js API Routes)
- **Database**: PostgreSQL (Supabase)
- **Authentication**: Supabase Auth
- **Storage**: Supabase Storage
- **Hosting**: Vercel (Frontend), Supabase (Backend)
- **Push Notifications**: Firebase Cloud Messaging (Sonra)

- ✅ Backend & Database mimarı
- ✅ Web panel temel yapısı
- ❌ Mobile app (Flutter - Sonra)
- ❌ Push notifications (FCM - Sonra)

## Database Şeması

### 11 Main Tables

```sql
profiles          -- Supabase Auth users (role bilgisi)
├── clinics        -- Veteriner klinikleri
│   ├── clinic_members     -- Clinic admin users
│   ├── owners            -- Pet owners/customers
│   │   └── pets          -- Hayvanlar
│   │       ├── appointments     -- Randevular
│   │       ├── health_records   -- Aşı/İlaç/Kontrol
│   │       ├── reminders        -- Hatırlatmalar
│   │       └── price_requests   -- Fiyat talepleri
│   ├── services    -- Hizmetler
│   └── invite_links -- QR/Davet kodları
```

### Enum Tipleri

```typescript

health_record_type = 'vaccination' | 'medication' | 'checkup'
```

## Güvenlik & RLS

### Row Level Security (RLS) Politikaları

Tüm tablolar RLS ile korunmaktadır. **her biri için SELECT, INSERT, UPDATE** politikaları mevcut.

#### Politika Hiyerarşisi

```
┌─────────────────────────────────────────┐
│          Super Admin                    │
│  → Tüm kliniklerin tüm verileri        │
│  → Sistem metriklerini görebilir       │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│       Clinic Admin (Per Clinic)         │
│  → Yalnız kendi kliniğinin verileri    │
│  → Müşteri, hayvan, randevu yönet      │
│  → Davet kodu oluştur                  │
│  → Sağlık kaydı gir                    │
│  → Fiyat talebi yanıtla                │
└─────────────────────────────────────────┘

┌─────────────────────────────────────────┐
│        Pet Owner                        │
│  → Yalnız kendi verilerine erişebilir  │
│  → Hayvanlarını görebilir              │
│  → Randevu talep edebilir              │
│  → Fiyat bilgisi isteyebilir           │
│  → Hatırlatmaları görebilir            │
└─────────────────────────────────────────┘
```

### Örnek RLS Politikaları

#### Clinic Admin - Owners Görüntüleme
```sql
-- Clinic admin sadece kendi kliniğinin owners'ını görebilir
CREATE POLICY "clinic_admin_view_clinic_owners" ON owners
FOR SELECT USING (
  clinic_id IN (
    SELECT clinic_id FROM clinic_members
    WHERE profile_id = auth.uid()
      AND role = 'clinic_admin'
      AND is_active = true
  )
);
```

#### Pet Owner - Kendi Verilerine Erişim
```sql
-- Pet owner sadece kendi bağlı olduğu owner kaydını görebilir
CREATE POLICY "pet_owner_view_own_owner" ON owners
FOR SELECT USING (
  profile_id = auth.uid()
  AND (SELECT role FROM profiles WHERE id = auth.uid()) = 'pet_owner'
);
```

#### Clinic Admin - Randevu İşlemleri
```sql
-- Clinic admin randevu güncelleyebilir
CREATE POLICY "clinic_admin_update_appointments" ON appointments
FOR UPDATE USING (
  clinic_id IN (
    SELECT clinic_id FROM clinic_members
    WHERE profile_id = auth.uid()
      AND role = 'clinic_admin'
      AND is_active = true
  )
);
```

## Kurulum & Deployment

### 1. Database Kurulumu

```bash
# 1. Supabase Project oluştur
# https://supabase.com → New project

# 2. SQL migration çalıştır
# SQL Editor → New Query → Paste 001_initial_schema.sql

# 3. Seed data ekle (test için)
# SQL Editor → New Query → Paste seed.sql
```

### 2. Supabase Configuration

```bash
# .env.local dosyasını oluştur
cp web/nextjs/.env.example web/nextjs/.env.local

# Supabase credentials ekle
NEXT_PUBLIC_SUPABASE_URL=https://your-project.supabase.co
NEXT_PUBLIC_SUPABASE_ANON_KEY=your-anon-key
```

### 3. Database Types Üret

```bash
# TypeScript için types oluştur
npm run supabase:gen-types
```

### 4. Local Development

```bash
cd web/nextjs
npm install
npm run dev
```

Açın: http://localhost:3000

### 5. Production Deployment

**Frontend (Vercel)**:
```bash

# Deploy et
vercel
```

**Database (Supabase)**:
- Supabase Hosting'de otomatik manage edilir
- Backups: Supabase tarafından yapılır
- Scaling: Supabase Pro/Business planlı

## Veri Akışı Örnekleri

### Senaryo 1: Pet Owner Randevu Talep Eder

```
1. Pet Owner login yapar (/login)
   → JWT token alır
   → Profile role'e göre yönlendirilir

2. Pet Owner dashboard'a gider (/pet-owner/dashboard)
   → Hayvanlarını görür (RLS filter: profile_id = current_user)

3. "Randevu Talep Et" tıklar
   → POST /appointments {pet_id, clinic_id, reason_for_visit}
   → Supabase RLS kontrol eder:
      - owner_id = current_user'ın owner.id mi?
      - status = 'pending' mi?
   → Database'e yazılır (status: pending)

4. Clinic Admin notification alır
   → /clinic/appointments'a gider
   → Bekleyen talepleri görür (RLS filter: clinic_id = current_clinic)
   → "Onayla" tıklar → status: approved
```

### Senaryo 2: Clinic Admin Fiyat Talebini Cevaplar

```
1. Clinic Admin dashboard'a girer
   → GET /clinic/price-requests
   → RLS filter: clinic_id = clinic_members.clinic_id

2. Bekleyen talepleri görür
   → status = 'pending'

3. Talep detayına tıklar
   → PATCH /price_requests {quote_price, quote_notes}
   → RLS kontrol: clinic_id uyuşuyor mu?
   → status: answered

4. Pet Owner notification alır
   → WhatsApp ile cevabın linkini alır
```

## Notlar & Best Practices

### Security
✅ Row Level Security aktif
✅ JWT tokens via Supabase Auth
✅ API requests HTTPS only
✅ Sensitive data (service role key) server-only

### Performance
✅ Indexed frequently queried columns
✅ Supabase caching strategies
✅ Next.js SSR/SSG kullanıldığında

### Database Maintenance
- Regular backups (Supabase handles)
- Monitor RLS policies quarterly
- Archive old data yearly

### Development Workflow
1. Branch from main
2. Write migrations
3. Test locally
4. Create PR
5. Review & merge
6. Deploy to Vercel
